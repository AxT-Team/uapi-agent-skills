# AES 解密

**接口地址**：`POST /text/aes/decrypt`

**分类**：[Text](../resources/Text.md)

**Operation ID**：`post-text-aes-decrypt`

## 这个接口适合什么时候用

收到了用AES加密的密文？把它、密钥和随机数（nonce）交给我们，就能还原出原始内容。

## 功能概述
这是一个标准的AES解密接口。你需要提供经过Base64编码的密文、加密时使用的密钥和nonce（随机数，通常为16字节字符串）。

> [!IMPORTANT]
> **关于密钥 `key`**
> 我们支持 AES-128, AES-192, 和 AES-256。请确保你提供的密钥 `key` 的长度（字节数）正好是 **16**、**24** 或 **32**，以分别对应这三种加密强度。
> 
> **关于随机数 `nonce`**
> 通常为16字节字符串，需与加密时一致。

## 调用前检查

- 先确认用户真正需要的是最终结果，而不是某个中间步骤。
- 如果参数说明里写了互斥、默认值或生效条件，请严格按说明组织请求。
- 如果用户没有提供必要参数，先补齐参数再调用，不要靠猜。

## 参数

这个接口没有显式参数，通常可以直接调用。

## 请求体

包含待解密文本 'text'、密钥 'key' 和随机数 'nonce' 的JSON对象

**是否必填**：是

**内容类型**：`application/json`

## 响应码

| 状态码 | 说明 |
|--------|------|
| `200` | 成功响应 |
| `400` | 无效的请求参数 |
| `500` | 解密失败 |
