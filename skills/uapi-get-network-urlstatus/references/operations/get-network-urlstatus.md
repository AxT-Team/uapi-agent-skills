# 检查URL的可访问性状态

**接口地址**：`GET /network/urlstatus`

**分类**：[Network](../resources/Network.md)

**Operation ID**：`get-network-urlstatus`

## 这个接口适合什么时候用

你的网站或API还好吗？用这个接口给它做个快速“体检”吧。

## 功能概述
提供一个URL，我们会向它发起一个请求，并返回其HTTP响应状态码。这是一种简单而有效的服务可用性监控方法。

## 调用前检查

- 先确认用户真正需要的是最终结果，而不是某个中间步骤。
- 如果参数说明里写了互斥、默认值或生效条件，请严格按说明组织请求。
- 如果用户没有提供必要参数，先补齐参数再调用，不要靠猜。

## 参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| `url` | query | string (url) | 是 | 你需要检查其可访问性状态的完整URL。 |

## 响应码

| 状态码 | 说明 |
|--------|------|
| `200` | 检查完成！根据目标URL的可达性返回不同结果。 |
| `400` | 请求参数无效。请检查 `url` 参数是否提供且格式正确。 |
| `502` | 请求URL失败（例如，DNS解析失败、连接超时） |
