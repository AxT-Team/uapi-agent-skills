# 敏感词分析 (GET)

**接口地址**：`GET /sensitive-word/analyze-query`

**分类**：[敏感词识别](../resources/敏感词识别.md)

**Operation ID**：`get-sensitive-word-analyze-query`

## 这个接口适合什么时候用

通过URL查询参数分析单个关键词，便于GET请求调用。

## 调用前检查

- 先确认用户真正需要的是最终结果，而不是某个中间步骤。
- 如果参数说明里写了互斥、默认值或生效条件，请严格按说明组织请求。
- 如果用户没有提供必要参数，先补齐参数再调用，不要靠猜。

## 参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| `keyword` | query | string | 是 | 要分析的关键词，最长50字符。 |

## 响应码

| 状态码 | 说明 |
|--------|------|
| `200` | 分析成功 |
| `400` | 请求参数错误 |
| `401` | 未授权访问 |

> 如果这个接口返回 `401`，先检查当前请求是不是缺少认证信息，或者使用了无效、过期的 Key / Token。
