# 答案之书 (POST)

**接口地址**：`POST /answerbook/ask`

**分类**：[Random](../resources/Random.md)

**Operation ID**：`post-answerbook-ask`

## 这个接口适合什么时候用

通过POST请求向答案之书提问并获得神秘答案。

## 功能概述
与GET方式相同，但通过JSON请求体发送问题，适合在需要发送较长问题或希望避免URL编码问题的场景中使用。

## 请求体格式
请求体必须是有效的JSON格式，包含question字段。

## 调用前检查

- 先确认用户真正需要的是最终结果，而不是某个中间步骤。
- 如果参数说明里写了互斥、默认值或生效条件，请严格按说明组织请求。
- 如果用户没有提供必要参数，先补齐参数再调用，不要靠猜。

## 参数

这个接口没有显式参数，通常可以直接调用。

## 请求体

包含问题的JSON对象

**是否必填**：是

**内容类型**：`application/json`

## 响应码

| 状态码 | 说明 |
|--------|------|
| `200` | 成功获取答案。 |
| `400` | 请求参数无效。 |
| `500` | 服务器内部错误。 |
