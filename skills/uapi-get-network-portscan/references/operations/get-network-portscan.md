# 端口扫描

**接口地址**：`GET /network/portscan`

**分类**：[Network](../resources/Network.md)

**Operation ID**：`get-network-portscan`

## 这个接口适合什么时候用

想检查一下你的服务器上某个端口（比如SSH的22端口或者Web的80端口）是否对外开放？这个工具可以帮你快速确认。

## 功能概述
你可以指定一个主机和端口号，我们的服务器会尝试连接该端口，并告诉你它是开放的（open）、关闭的（closed）还是超时了（timeout）。这对于网络服务配置检查和基本的安全扫描很有用。

## 调用前检查

- 先确认用户真正需要的是最终结果，而不是某个中间步骤。
- 如果参数说明里写了互斥、默认值或生效条件，请严格按说明组织请求。
- 如果用户没有提供必要参数，先补齐参数再调用，不要靠猜。

## 参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| `host` | query | string | 是 | 需要扫描的目标主机，可以是域名或IP地址。 |
| `port` | query | integer | 是 | 需要扫描的端口号，范围是 1 到 65535。 |
| `protocol` | query | enum: tcp, udp | 否 | 扫描使用的协议，可以是 'tcp' 或 'udp'。 |

## 响应码

| 状态码 | 说明 |
|--------|------|
| `200` | 扫描完成！返回端口的状态。 |
| `400` | 请求参数无效。请检查 `host`, `port` 等参数是否提供且格式正确。 |
| `500` | 扫描失败。服务器在执行扫描时发生内部错误。 |
